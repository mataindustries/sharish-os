## Product rules
- mobile first
- dark carbon clinical ops aesthetic
- no gradient startup styling
- no oversized rounded cards
- hard-edged slab panels
- huge current-step typography
- giant glove-safe Next Step button
- doctor-specific preference injection
- local JSON seed data only for pass 1
- no auth
- no backend dependency for pass 1
