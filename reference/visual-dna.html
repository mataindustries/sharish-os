Visual DNA Rules
Build Spec
This UI should feel like a live operatory system, not a generic SaaS dashboard. Action lives in the center. Readiness and exceptions live in the side rails. Color is sparse and purposeful: cyan for now, cobalt for context, amber or red for pay-attention states.
No soft startup UI. Avoid purple gradients, bubbly cards, playful chat layouts, or oversized rounded corners.
One dominant action. Every critical screen needs one massive primary control that works with gloves and pressure.
Doctor-specific injection. Preference notes, timing quirks, and material swaps must appear in structured places, never hidden in paragraphs.
State over decoration. The interface should clearly communicate ready, missing, current, next, blocked, or complete.

Type + Interaction
Rules
Typeface split. Inter for critical instruction. IBM Plex Mono for metadata, timers, step IDs, and doctor/procedure labels.
Geometry. Sharp slabs. 4px radius max. 1px borders. Hard depth only.
Motion. No decorative float. Motion only for state change, listening state, or completion feedback.
